## End-to-End: Idempotent Re-crawl

**Purpose:** Validate that crawling the same unchanged directory tree twice produces identical registry state and manifest files.

1. Set up a fresh directory tree with 2 deliveries (1 passed, 1 pending) each containing 2 .sas7bdat files
2. Start registry API with a fresh database
3. Run crawler -- note manifest file contents and POST count
4. Run crawler again without changing anything
5. **Verify:** Manifest files have identical delivery_id, fingerprint, parsed, files, file_count fields. POST payloads have identical fingerprint values.

## End-to-End: Multi-Root Crawl

**Purpose:** Validate crawler processes deliveries across multiple scan_roots in a single run.

1. Create two separate scan root trees (e.g., `/tmp/root_qa/` and `/tmp/root_qm/`) each with one delivery
2. Configure both in scan_roots config
3. Run crawler
4. **Verify:** Both deliveries appear in registry, manifests written for both, scan_root field correct in each manifest
